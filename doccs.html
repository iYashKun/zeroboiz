tg Userbot API Documentation

Complete A-Z documentation for the Python/Flask/Pyrogram userbot API.

Overview

This repository exposes a Telegram userbot-style API using Flask and Pyrogram. The API supports session persistence, reconnect handling, chat joins, message fetch, media download, and media streaming.

Everything runs from app.py, with command routing defined in commands.py.

Repository Structure

  • app.py — main Flask API service and Pyrogram client management
  • commands.py — simple command router for /ping, /help, and /me
  • requirements.txt — Python dependencies
  • tests/test_app.py — API endpoint and helper tests
  • sessions/ — persistent Pyrogram session files
  • docs.html — this documentation page

Setup

Follow these steps to get the project running:

  1. Create a virtual environment:
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

Create a .env file in the project root with the following variables:

TG_API_ID=123456
TG_API_HASH=your_api_hash_here
SESSION_STRING=your_session_string_here
TG_SESSION_NAME=tg_userbot
TG_DEVICE_MODEL=TG-API
TG_APP_VERSION=1.0.0
PORT=8991

Note: SESSION_STRING is optional but recommended. If missing, session files are stored automatically under sessions/.

Run the App

python3 app.py

The service listens on port 8991 by default. You can change this with PORT in .env.

Available Endpoints

These are the exposed API routes in the current implementation:

  • GET /health — check service status
  • GET /api/join/<username|id|link> — join a Telegram chat or channel
  • GET /api/<username|id>/<message_id> — fetch a message by chat identifier and message ID
  • GET /api/v2/<chat_id>/<message_id> — legacy-style message fetch by numeric chat ID
  • POST /api/send — send a text message to a chat
  • POST /api/command — execute bot commands
  • GET /api/download/<username|id>/<message_id> — download message media to disk
  • GET /api/stream/<username|id>/<message_id> — stream downloaded media directly over HTTP

Endpoint Details

GET /health

Returns a JSON payload with service status, readiness, and configuration state.

{
  "status": "ok",
  "service": "tg-pyrogram-api",
  "ready": true,
  "configured": true,
  "message": "Userbot is ready to connect when credentials are provided."
}

GET /api/join/<identifier>

Joins a chat or channel by username, numeric ID, or Telegram invite link.

curl http://127.0.0.1:8991/api/join/@sageairdrops

Response example:

{
  "ok": true,
  "joined": true,
  "identifier": "@sageairdrops",
  "chat": "sageairdrops"
}

GET /api/<identifier>/<message_id>

Fetches a message from any chat identifier type, including username, link, or numeric ID.

curl http://127.0.0.1:8991/api/sageairdrops/123

GET /api/v2/<chat_id>/<message_id>

Fetches a message using a numeric chat ID path. This route preserves compatibility with earlier formats.

curl http://127.0.0.1:8991/api/v2/-1001234567890/123

POST /api/send

Sends text to a chat.

curl -X POST http://127.0.0.1:8991/api/send \
  -H "Content-Type: application/json" \
  -d '{"chat_id":"@sageairdrops","text":"Hello from userbot"}'

POST /api/command

Executes a supported bot-style command against the local router.

curl -X POST http://127.0.0.1:8991/api/command \
  -H "Content-Type: application/json" \
  -d '{"text":"/ping"}'

Built-in commands:

  • /ping — returns pong
  • /help — lists available commands
  • /me — returns bot readiness and chat context

Example response:

{
  "ok": true,
  "handled": true,
  "response": "pong"
}

GET /api/download/<identifier>/<message_id>

Downloads supported media from a message to the downloads/ directory and returns the saved path.

curl http://127.0.0.1:8991/api/download/@sageairdrops/123

Example response:

{
  "ok": true,
  "message_id": 123,
  "chat_id": "-100...",
  "file_path": "/workspaces/tg/downloads/123_photo.jpg",
  "file_name": "123_photo.jpg"
}

GET /api/stream/<identifier>/<message_id>

Streams the downloaded media file directly over HTTP. This is useful for embedding or previewing media without manual file handling.

curl http://127.0.0.1:8991/api/stream/@sageairdrops/123

Session Persistence & Auto-Reconnect

The service stores Pyrogram session data under sessions/ when SESSION_STRING is not supplied. That means the login session remains persistent across restarts.

The app also uses a dedicated event loop thread for Pyrogram and automatically reconnects when an endpoint triggers ensure_connected().

When to use SESSION_STRING

  • If you want headless startup without interactive login, provide a valid Telegram session string.
  • If you omit it, the app saves session files automatically under sessions/.

Common Troubleshooting

Service says not configured

Make sure TG_API_ID and TG_API_HASH are present in .env and properly loaded by the app.

Loop mismatch error

The implementation uses a dedicated Pyrogram event loop thread to avoid errors like Future attached to a different loop.

File not found after download

Make sure the message contains media and the service has permission to write to downloads/.

Testing

Run the repository tests with:

python3 -m unittest discover -s tests

The test suite covers health checks, endpoint errors, command handling, and media route behavior.
